# 缓存判定与时效性规则

## 时效性分级

| 级别 | 特征 | 策略 |
|------|------|------|
| **低** | 不随时间变化 | 7 天内直接用 |
| **中** | 可能变化但周期长 | 24h 内直接用；超期先复用再验证 |
| **高** | 需要当前最新数据 | **必须重新搜索**，缓存仅供背景参考 |
| **验证** | 用户要求确认旧结论 | 始终重新搜索，新旧对比 |

用户问题含「最新」「现在」「实时」「今日」→ 直接按高时效处理。不确定时按高时效。

## 混合查询

多子问题拆分时效性。高时效走搜索，低时效从缓存或训练数据回答，最终合并。

## 判定步骤

1. `stat` 或 `ls -la` 查文件修改时间
2. 对照上表分级
3. 低时效 → 直接读文件回复
4. 中时效 → 窗口内复用，超期简单验证
5. 高时效 → 跳到「环境检查」开始完整搜索

## 保留策略

输出文件保留 7 天，check-deps.mjs 自动清理。手动清理：

```bash
node "${CLAUDE_SKILL_DIR}/../../scripts/cleanup-output.mjs" [--days N] [--dry-run]
```
